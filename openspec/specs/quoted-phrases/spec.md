# Quoted Phrases Specification

> Generated by spec-gen on 2026-04-05
> Source files: `Sources/SearchExpressionParser/Tokenization/TokenExtractor/PhraseExtractor.swift`, `Tests/SearchExpressionParserTests/TokenizerTests.swift`

## Purpose

The quoted phrase tokenizer recognizes double-quoted strings in search input and produces `Phrase` tokens that preserve internal whitespace verbatim. This allows users to search for exact multi-word sequences, distinguishing them from separate `Word` tokens that would otherwise be split on whitespace.

## Requirements

### Requirement: Opening and Closing Delimiters

The system SHALL recognize `"` as the phrase opening delimiter. The next unescaped `"` in the input SHALL close the phrase. All characters between the delimiters SHALL be captured as the content of a `Phrase` token.

#### Scenario: Single quoted word

- **GIVEN** the input `"justice"`
- **WHEN** the tokenizer processes the input
- **THEN** it SHALL produce `Phrase("justice")`

#### Scenario: Quoted phrase with multiple words

- **GIVEN** the input `"  fair   play "`
- **WHEN** the tokenizer processes the input
- **THEN** it SHALL produce `Phrase("  fair   play ")`

### Requirement: Whitespace Preservation

The system SHALL preserve all internal whitespace verbatim within a quoted phrase. Whitespace MUST NOT be collapsed or trimmed.

#### Scenario: Leading, trailing, and multiple internal spaces

- **GIVEN** the input `"  fair   play "`
- **WHEN** the tokenizer processes the input
- **THEN** the resulting `Phrase` content SHALL be `  fair   play ` with all spaces preserved exactly

### Requirement: Escaped Quotation Marks

The system SHALL recognize `\"` as an escape sequence inside a quoted phrase. The escape sequence SHALL produce a literal `"` character in the phrase content. Only `\"` is recognized as an escape inside quotes; `\\` is NOT treated as an escape sequence within phrases.

#### Scenario: Escaped quote within phrase

- **GIVEN** the input `"foo   \"  bar"`
- **WHEN** the tokenizer processes the input
- **THEN** it SHALL produce `Phrase("foo   \"  bar")`

### Requirement: Lone Quotation Mark

When a `"` appears at the end of input with no subsequent characters, the system SHALL produce a `Word("\"")` token instead of a `Phrase`.

#### Scenario: Single quotation mark at end of input

- **GIVEN** the input `"`
- **WHEN** the tokenizer processes the input
- **THEN** it SHALL produce `Word("\"")`

### Requirement: Unclosed Quoted Phrase

When a `"` opens a phrase but no closing `"` is found before end of input, the system SHALL produce a `Phrase` token containing all characters after the opening quote. The content is NOT treated as a `Word`.

#### Scenario: Quote followed by spaces with no closing quote

- **GIVEN** the input `"   ` (quote followed by spaces, no closing quote)
- **WHEN** the tokenizer processes the input
- **THEN** it SHALL produce `Phrase("   ")`

### Requirement: Empty Quoted Phrase

When two consecutive `"` characters appear (`""`), the system SHALL produce a `Phrase("")` token with empty string content.

#### Scenario: Empty double quotes

- **GIVEN** the input `""`
- **WHEN** the tokenizer processes the input
- **THEN** it SHALL produce `Phrase("")`
