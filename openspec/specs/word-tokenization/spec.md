# Word Tokenization Specification

> Generated by spec-gen on 2026-04-05
> Source files: `Sources/SearchExpressionParser/Tokenization/TokenExtractor/WordExtractor.swift`, `Sources/SearchExpressionParser/Tokenization/Tokens.swift`

## Purpose

WordExtractor tokenizes contiguous sequences of non-whitespace, non-parenthesis, non-quotation-mark characters into `Word` tokens. It serves as the fallback (wildcard) extractor in the tokenizer pipeline, tried last after all other extractors fail to match. `Word` is a typealias for `Phrase`, so both types share the same underlying struct.

## Requirements

### Requirement: Word Character Set

The system SHALL treat a word as a maximal sequence of characters that are not whitespace, not parentheses (`(` or `)`), and not quotation marks (`"`).

#### Scenario: Simple word

- **GIVEN** the input string `foo`
- **WHEN** the tokenizer processes the input
- **THEN** it produces a single `Word("foo")` token

#### Scenario: Punctuation preserved within words

- **GIVEN** the input string `amet!`
- **WHEN** the tokenizer processes the input
- **THEN** it produces a single `Word("amet!")` token, preserving the trailing punctuation

#### Scenario: Question mark preserved within words

- **GIVEN** the input string `Consectetur?`
- **WHEN** the tokenizer processes the input
- **THEN** it produces a single `Word("Consectetur?")` token

### Requirement: Whitespace Splits Words

The system SHALL split words at whitespace boundaries. Leading and trailing whitespace SHALL be ignored.

#### Scenario: Two words separated by whitespace

- **GIVEN** the input string `   foo   bar   `
- **WHEN** the tokenizer processes the input
- **THEN** it produces `[Word("foo"), Word("bar")]`

#### Scenario: Single character with surrounding whitespace

- **GIVEN** the input string `     x    `
- **WHEN** the tokenizer processes the input
- **THEN** it produces `[Word("x")]`

### Requirement: Parentheses Break Word Boundaries

The system SHALL treat `(` and `)` as word boundary delimiters. A parenthesis character terminates the current word and is extracted as its own token by the parentheses extractors.

#### Scenario: Word inside parentheses

- **GIVEN** the input string `(red)`
- **WHEN** the tokenizer processes the input
- **THEN** it produces `[OpeningParens(), Word("red"), ClosingParens()]`

#### Scenario: Words adjacent to closing parentheses

- **GIVEN** the input string `warrior) what`
- **WHEN** the tokenizer processes the input
- **THEN** the word `warrior` terminates before `)`, producing `[Word("warrior"), ClosingParens(), Word("what")]`

### Requirement: Quotation Marks Break Word Boundaries

The system SHALL treat `"` as a word boundary delimiter. An unescaped quotation mark terminates the current word and begins phrase extraction via PhraseExtractor.

#### Scenario: Quoted phrase after words

- **GIVEN** the input string `called "  another "`
- **WHEN** the tokenizer processes the input
- **THEN** the word `called` is separate from the phrase, producing `[Word("called"), Phrase("  another ")]`

### Requirement: Escape Sequences

The system SHALL interpret `\` as an escape character within words. When `\` is followed by any character, the `\` is consumed and the following character is included literally in the word. When `\` appears at end of input with no following character, the `\` itself is included in the word.

#### Scenario: Escaped letter within a word

- **GIVEN** the input string `you\know`
- **WHEN** the tokenizer processes the input
- **THEN** the `\` is consumed and it produces `[Word("youknow")]`

#### Scenario: Escaped backslash

- **GIVEN** the input string `\\`
- **WHEN** the tokenizer processes the input
- **THEN** it produces `[Word("\")]`, a word containing a single literal backslash

#### Scenario: Escaped backslash within a word

- **GIVEN** the input string `a\\b`
- **WHEN** the tokenizer processes the input
- **THEN** it produces `[Word("a\b")]`

#### Scenario: Escaped parenthesis

- **GIVEN** the input string `\(`
- **WHEN** the tokenizer processes the input
- **THEN** it produces `[Word("(")]`, treating the parenthesis as a literal character

#### Scenario: Escaped quotation mark

- **GIVEN** the input string `\"`
- **WHEN** the tokenizer processes the input
- **THEN** it produces `[Word("\"")]`, treating the quotation mark as a literal character

#### Scenario: Trailing backslash at end of input

- **GIVEN** the input string `\`
- **WHEN** the tokenizer processes the input
- **THEN** it produces `[Word("\")]`, including the backslash literally

#### Scenario: Escaped backslash followed by letter

- **GIVEN** the input string `\\a`
- **WHEN** the tokenizer processes the input
- **THEN** it produces `[Word("\a")]`, a backslash followed by the letter

### Requirement: Fallback Extractor Ordering

The system SHALL try WordExtractor last in the extractor pipeline, after OpeningParensExtractor, ClosingParensExtractor, PhraseExtractor, BangExtractor, NotExtractor, AndExtractor, and OrExtractor. WordExtractor serves as the wildcard fallback.

#### Scenario: Extractor ordering <!-- @nocover: structural property, not runtime behavior -->

- **GIVEN** the default tokenizer configuration
- **WHEN** the extractor list is initialized
- **THEN** WordExtractor is the last entry in the list

### Requirement: Word Is Typealias for Phrase

The system SHALL define `Word` as a public typealias for `Phrase`. Both names refer to the same struct type, which conforms to `Token` and holds a single `string` property.

#### Scenario: Type equivalence <!-- @nocover: compile-time typealias, not runtime behavior -->

- **GIVEN** a `Word("hello")` and a `Phrase("hello")`
- **WHEN** compared
- **THEN** they are the same type and equal in value

### Requirement: Precondition Check

WordExtractor SHALL only attempt extraction when the next character in the buffer is not whitespace. The `matchesPreconditions` method returns `true` when the next character is non-whitespace, and `false` otherwise.

#### Scenario: Whitespace at current position <!-- @nocover: internal precondition, not public API behavior -->

- **GIVEN** a buffer positioned at a whitespace character
- **WHEN** `matchesPreconditions` is checked
- **THEN** it returns `false` and WordExtractor does not attempt extraction
