# Phrase Extraction Specification

> Generated by spec-gen on 2026-04-05
> Source files: `Sources/SearchExpressionParser/NormalForm/PhraseCollectionConvertible.swift`, `Sources/SearchExpressionParser/NormalForm/ContainmentEvaluator.swift`

## Purpose

Phrase extraction collects candidate search terms from a parsed expression tree for use in highlighting matches in UI. It returns all positive (non-negated) terms that *may* appear in the haystack, treating OR branches as alternatives that all contribute candidates. Normalization to negation normal form is applied first so that NOT nodes only wrap leaf nodes and produce no phrases.

## Requirements

### Requirement: ContainsNode contributes its string as a phrase

The system SHALL return a single-element array containing the node's string value when collecting phrases from a `ContainsNode`.

#### Scenario: Single contains term
- **GIVEN** a `ContainsNode` with string `"foo"`
- **WHEN** phrases are collected
- **THEN** the result is `["foo"]`

### Requirement: AnythingNode contributes no phrases

The system SHALL return an empty array when collecting phrases from an `AnythingNode`.

#### Scenario: Wildcard match
- **GIVEN** an `AnythingNode`
- **WHEN** phrases are collected
- **THEN** the result is `[]`

### Requirement: NotNode contributes no phrases

The system SHALL return an empty array when collecting phrases from a `NotNode`, regardless of its child expression. This relies on normalization having been applied first so that NOT only wraps leaf nodes.

#### Scenario: Negated term
- **GIVEN** a `NotNode` wrapping `ContainsNode("foo")`
- **WHEN** phrases are collected
- **THEN** the result is `[]`

### Requirement: AndNode concatenates phrases from both children

The system SHALL return the concatenation of phrases from the left-hand side followed by phrases from the right-hand side when collecting phrases from an `AndNode`.

#### Scenario: Two positive terms
- **GIVEN** an `AndNode` with `ContainsNode("foo")` and `ContainsNode("bar")`
- **WHEN** phrases are collected
- **THEN** the result is `["foo", "bar"]`

#### Scenario: One negated child
- **GIVEN** an `AndNode` with `NotNode(ContainsNode("foo"))` and `ContainsNode("bar")`
- **WHEN** phrases are collected via `ContainmentEvaluator`
- **THEN** the result is `["bar"]`

### Requirement: OrNode concatenates phrases from both children

The system SHALL return the concatenation of phrases from both branches of an `OrNode`. Phrases are "may contain" candidates, not "must contain", so OR branches contribute all alternatives.

#### Scenario: Two alternative terms
- **GIVEN** an `OrNode` with `ContainsNode("foo")` and `ContainsNode("bar")`
- **WHEN** phrases are collected
- **THEN** the result is `["foo", "bar"]`

#### Scenario: One negated alternative
- **GIVEN** an `OrNode` with `ContainsNode("foo")` and `NotNode(ContainsNode("bar"))`
- **WHEN** phrases are collected via `ContainmentEvaluator`
- **THEN** the result is `["foo"]`

### Requirement: Runtime type casting for child nodes

The system SHALL use runtime type casting (`as? PhraseCollectionConvertible`) when accessing child nodes of `AndNode` and `OrNode`. If a child does not conform, its phrases default to an empty array.

#### Scenario: Non-conforming child
- **GIVEN** an `AndNode` or `OrNode` with a child that does not conform to `PhraseCollectionConvertible`
- **WHEN** phrases are collected
- **THEN** the non-conforming child contributes `[]`

### Requirement: ContainmentEvaluator normalizes before collecting phrases

The system SHALL normalize the expression to negation normal form before collecting phrases. `ContainmentEvaluator.phrases()` calls `normalizedEvaluable()` (which pushes negations inward via De Morgan's laws) and then collects phrases from the result.

#### Scenario: Negated AND expression
- **GIVEN** `NOT(x AND y)` as input to `ContainmentEvaluator`
- **WHEN** `phrases()` is called
- **THEN** normalization produces `NOT(x) OR NOT(y)`, and phrases returns `[]`

### Requirement: ContainmentEvaluator returns empty array on recursion overflow

The system SHALL return an empty array from `phrases()` if normalization throws `RecursionTooDeepError` due to exceeding `maxRecursion` depth (default 50).

#### Scenario: Deeply nested expression
- **GIVEN** an expression that exceeds the recursion limit
- **WHEN** `phrases()` is called
- **THEN** the result is `[]`
