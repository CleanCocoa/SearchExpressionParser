# Negation Normal Form Specification

> Generated by spec-gen on 2026-04-05
> Source files: Sources/SearchExpressionParser/NormalForm/ContainmentEvaluator.swift, Tests/SearchExpressionParserTests/ContainmentEvaluatorTests.swift

## Purpose

`ContainmentEvaluator` transforms expression trees into negation normal form (NNF) by pushing `NOT` nodes inward using De Morgan's laws until negations only appear on leaf nodes. This normalization enables extraction of positive phrases from an expression for uses like match highlighting. The `phrases()` method returns the set of string values that may be contained in a match, excluding negated terms.

## Requirements

### Requirement: Non-NOT nodes returned unchanged

The system SHALL return any expression node that is not a `NotNode` unchanged from `pushNegation`.

#### Scenario: Leaf nodes pass through

- **GIVEN** an `AnythingNode` or `ContainsNode` expression
- **WHEN** normalization is performed via `normalizedEvaluable()`
- **THEN** the original expression is returned unchanged

### Requirement: NOT over AND applies De Morgan's law

The system SHALL transform `NOT(A AND B)` into `OR(NOT(A), NOT(B))`, recursively normalizing each operand.

#### Scenario: NOT wrapping an AND of two leaf nodes

- **GIVEN** an expression `NOT(AND(contains("x"), contains("y")))`
- **WHEN** normalization is performed
- **THEN** the result is `OR(NOT(contains("x")), NOT(contains("y")))`

### Requirement: NOT over OR applies De Morgan's law

The system SHALL transform `NOT(A OR B)` into `AND(NOT(A), NOT(B))`, recursively normalizing each operand.

#### Scenario: NOT wrapping an OR of two leaf nodes

- **GIVEN** an expression `NOT(OR(contains("x"), contains("y")))`
- **WHEN** normalization is performed
- **THEN** the result is `AND(NOT(contains("x")), NOT(contains("y")))`

### Requirement: NOT over leaf nodes preserved

The system SHALL return `NOT(leaf)` unchanged when the inner expression is neither an `AndNode` nor an `OrNode`.

#### Scenario: NOT wrapping a ContainsNode

- **GIVEN** an expression `NOT(contains("x"))`
- **WHEN** normalization is performed
- **THEN** the result is `NOT(contains("x"))` unchanged

### Requirement: Multi-level normalization through arbitrary nesting

The system SHALL recursively apply De Morgan's laws through multiple levels of nesting until all negations reach leaf nodes.

#### Scenario: Two levels of nesting

- **GIVEN** an expression `NOT(AND(OR(contains("a"), contains("b")), AND(contains("c"), contains("d"))))`
- **WHEN** normalization is performed
- **THEN** the result is `OR(AND(NOT(contains("a")), NOT(contains("b"))), OR(NOT(contains("c")), NOT(contains("d"))))`

### Requirement: Recursion depth guard

The system SHALL soft-deprecate the recursion depth guard. `maxRecursion` SHALL be accepted but ignored. `RecursionTooDeepError` SHALL be marked as deprecated and never thrown. `normalizedEvaluable()` SHALL remain `throws` for source compatibility but SHALL NOT throw.

#### Scenario: Default initialization no longer enforces limit

- **GIVEN** a `ContainmentEvaluator` initialized with default parameters
- **WHEN** the expression tree depth exceeds 50 levels during normalization
- **THEN** normalization SHALL complete successfully without throwing

#### Scenario: Custom recursion limit ignored

- **GIVEN** a `ContainmentEvaluator` initialized with `maxRecursion: 5`
- **WHEN** the expression tree depth exceeds 5 during normalization
- **THEN** normalization SHALL complete successfully without throwing

### Requirement: Phrases extraction excludes negated terms

The system SHALL return only non-negated leaf phrases from the normalized expression via the `phrases()` method.

#### Scenario: AND with one negated operand

- **GIVEN** an expression `AND(contains("foo"), NOT(contains("bar")))`
- **WHEN** `phrases()` is called
- **THEN** the result is `["foo"]`
