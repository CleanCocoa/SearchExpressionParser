## Purpose

SearchExpressionParser is a Swift package that converts user-provided search strings into evaluable boolean expression trees. It supports boolean operators (AND, OR, NOT), parenthesized grouping, and quoted phrases, producing a tree of expression nodes that can be evaluated against any string to determine whether it satisfies the search criteria. The package targets macOS 10.13+ and requires Swift 5.5+.

## Requirements

### Requirement: Public Entry Point

The system SHALL expose a static method `Parser.parse(searchString:)` that accepts a search string and returns an `Expression` tree. This method SHALL perform tokenization followed by parsing in a single call.

#### Scenario: Parse a simple search string

- **GIVEN** a search string `"foo bar"`
- **WHEN** `Parser.parse(searchString:)` is called
- **THEN** an expression tree representing `AND(contains("foo"), contains("bar"))` SHALL be returned

### Requirement: Tokenization

The system SHALL tokenize a search string into a sequence of `Token` values. The tokenizer SHALL recognize the following token types: `Word` (bare terms), `Phrase` (quoted strings), `OpeningParens`, `ClosingParens`, `BinaryOperator.and`, `BinaryOperator.or`, `UnaryOperator.not`, and `UnaryOperator.bang`. Whitespace SHALL be consumed as a delimiter and not produce tokens.

#### Scenario: Tokenize a string with operators

- **GIVEN** the search string `"foo AND NOT bar"`
- **WHEN** the `Tokenizer` produces tokens
- **THEN** the token sequence SHALL be `[Word("foo"), BinaryOperator.and, UnaryOperator.not, Word("bar")]`

#### Scenario: Tokenize a quoted phrase

- **GIVEN** the search string `"\"hello world\""`
- **WHEN** the `Tokenizer` produces tokens
- **THEN** a single `Phrase("hello world")` token SHALL be produced

### Requirement: Parentheses Balancing

The system SHALL balance unmatched parentheses before parsing. Unmatched opening parentheses SHALL be converted to `Word("(")` tokens. Unmatched closing parentheses SHALL be converted to `Word(")")` tokens.

#### Scenario: Unmatched closing parenthesis

- **GIVEN** the token sequence `[Word("foo"), ClosingParens]`
- **WHEN** parentheses balancing runs
- **THEN** the closing parenthesis SHALL be replaced with `Word(")")`

### Requirement: Expression Tree Construction

The parser SHALL produce an expression tree composed of five node types:

- `AnythingNode`: wildcard that matches any input. SHALL be produced when the token stream is empty.
- `ContainsNode`: leaf node matching a literal string. SHALL store both the original `String` and a precomputed `CString` (lowercased, precomposed UTF-8).
- `NotNode`: unary negation wrapping one child expression.
- `AndNode`: binary conjunction of two child expressions.
- `OrNode`: binary disjunction of two child expressions.

#### Scenario: Empty search string

- **GIVEN** an empty search string `""`
- **WHEN** the parser produces an expression
- **THEN** an `AnythingNode` SHALL be returned

#### Scenario: Implicit AND between adjacent terms

- **GIVEN** the search string `"foo bar"`
- **WHEN** the parser produces an expression
- **THEN** an `AndNode` SHALL be returned with `ContainsNode("foo")` on the left and `ContainsNode("bar")` on the right

#### Scenario: Negation with NOT keyword

- **GIVEN** the search string `"NOT foo"`
- **WHEN** the parser produces an expression
- **THEN** a `NotNode` wrapping `ContainsNode("foo")` SHALL be returned

#### Scenario: Negation with bang operator

- **GIVEN** the search string `"!foo"`
- **WHEN** the parser produces an expression
- **THEN** a `NotNode` wrapping `ContainsNode("foo")` SHALL be returned

#### Scenario: OR operator

- **GIVEN** the search string `"foo OR bar"`
- **WHEN** the parser produces an expression
- **THEN** an `OrNode` SHALL be returned with `ContainsNode("foo")` on the left and `ContainsNode("bar")` on the right

#### Scenario: Parenthesized grouping

- **GIVEN** the search string `"(foo OR bar) baz"`
- **WHEN** the parser produces an expression
- **THEN** an `AndNode` SHALL be returned whose left child is `OrNode(ContainsNode("foo"), ContainsNode("bar"))` and whose right child is `ContainsNode("baz")`

### Requirement: Expression Evaluation

Each expression node SHALL implement the `Expression` protocol, providing two evaluation methods:

- `isSatisfied(by:)` accepting a `StringExpressionSatisfiable` for literal `String.contains` matching.
- `isSatisfied(by:)` accepting a `CStringExpressionSatisfiable` for `CString`-based matching via lowercased, precomposed UTF-8 comparison.

`String` SHALL conform to `StringExpressionSatisfiable` by default.

#### Scenario: Evaluate an AND expression

- **GIVEN** an `AndNode(ContainsNode("foo"), ContainsNode("bar"))` expression
- **WHEN** evaluated against a `StringExpressionSatisfiable` containing `"foo bar"`
- **THEN** `isSatisfied(by:)` SHALL return `true`

#### Scenario: Evaluate a NOT expression

- **GIVEN** a `NotNode(ContainsNode("baz"))` expression
- **WHEN** evaluated against a `StringExpressionSatisfiable` containing `"foo"`
- **THEN** `isSatisfied(by:)` SHALL return `true`

### Requirement: Phrase Extraction and Normalization

The system SHALL provide a `ContainmentEvaluator` that extracts positive search phrases from an expression tree. It SHALL apply negation normal form (De Morgan's laws) to push `NotNode` wrappers inward before extraction. Phrases inside `NotNode` branches SHALL be excluded from results. The evaluator SHALL accept a `maxRecursion` parameter (default 50) and return an empty array if normalization exceeds that depth.

All five node types SHALL conform to `PhraseCollectionConvertible`, providing a `phrases` property that collects contained literal strings.

#### Scenario: Extract phrases excluding negated terms

- **GIVEN** the expression tree for `"foo AND bar AND NOT baz"`
- **WHEN** `ContainmentEvaluator.phrases()` is called
- **THEN** the result SHALL be `["foo", "bar"]`

#### Scenario: Recursion limit exceeded

- **GIVEN** an expression tree whose normalization requires more than `maxRecursion` levels
- **WHEN** `ContainmentEvaluator.phrases()` is called
- **THEN** the result SHALL be an empty array

## Technical Notes

- **Implementation**: `Sources/SearchExpressionParser/`
- **Platform**: macOS 10.13+ (Swift tools version 5.5)
- **Pipeline**: `Tokenizer` -> `balanceParentheses` -> `Parser` -> `Expression` tree -> evaluation via `isSatisfied(by:)` or normalization via `ContainmentEvaluator`
- **Dependencies**: None (standalone Swift package)
- **CString optimization**: `ContainsNode` precomputes a lowercased, precomposed UTF-8 `CString` for fast `strstr`-style matching via `CStringExpressionSatisfiable`
- **Operator precedence**: Implicit AND binds adjacent terms; explicit AND and OR are parsed left-to-right via recursive descent
