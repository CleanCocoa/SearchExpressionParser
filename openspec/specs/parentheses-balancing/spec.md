# Parentheses Balancing Specification

> Generated by spec-gen on 2026-04-05
> Source files: Sources/SearchExpressionParser/Parsing/Parser.swift, Tests/SearchExpressionParserTests/BalanceParenthesesTests.swift

## Purpose

Parentheses balancing is a pre-parse transformation applied to the token array before the recursive descent parser runs. It ensures every `OpeningParens` token has a matching `ClosingParens` token, converting unmatched parentheses into `Phrase` (aka `Word`) tokens containing the literal character. This guarantees the parser never encounters unbalanced grouping tokens.

## Requirements

### Requirement: Pre-Parse Transformation

The system SHALL apply parentheses balancing to the token array during `Parser.init`, before any parsing occurs. The `balanceParentheses` function receives the raw token array and returns a transformed array where all remaining `OpeningParens`/`ClosingParens` tokens are guaranteed to be properly paired. The implementation SHALL use O(1) call stack depth regardless of input size or nesting depth.

#### Scenario: Non-Parenthesis Tokens Pass Through Unchanged

- **GIVEN** a token array containing only non-parenthesis tokens (e.g., `[Phrase("asd"), Phrase("def")]`)
- **WHEN** balanceParentheses is applied
- **THEN** the token array is returned unchanged

#### Scenario: Empty Input

- **GIVEN** an empty token array
- **WHEN** balanceParentheses is applied
- **THEN** an empty token array is returned

### Requirement: Matched Pairs Preserved

The system SHALL preserve `OpeningParens` and `ClosingParens` tokens when they form properly matched pairs.

#### Scenario: Simple Balanced Pair

- **GIVEN** the token array `[OpeningParens, Word("some"), BinaryOperator.or, Word("stuff"), ClosingParens]`
- **WHEN** balanceParentheses is applied
- **THEN** the token array is returned unchanged

#### Scenario: Empty Balanced Pair

- **GIVEN** the token array `[OpeningParens, ClosingParens]`
- **WHEN** balanceParentheses is applied
- **THEN** the token array is returned unchanged with both tokens preserved as `OpeningParens` and `ClosingParens`

### Requirement: Unmatched Opening Parens Converted to Phrase

The system SHALL convert any `OpeningParens` token that has no matching `ClosingParens` into a `Phrase("(")` token (equivalent to `Word("(")`).

#### Scenario: Single Unmatched Opening Paren

- **GIVEN** the token array `[OpeningParens]`
- **WHEN** balanceParentheses is applied
- **THEN** the result is `[Phrase("(")]`

#### Scenario: Excess Opening Paren With Balanced Pair

- **GIVEN** the token array `[OpeningParens, Word("some"), OpeningParens, ClosingParens]`
- **WHEN** balanceParentheses is applied
- **THEN** the first `OpeningParens` (unmatched) becomes `Phrase("(")`, while the inner pair remains: `[Phrase("("), Word("some"), OpeningParens, ClosingParens]`

### Requirement: Unmatched Closing Parens Converted to Phrase

The system SHALL convert any `ClosingParens` token that has no matching `OpeningParens` into a `Phrase(")")` token (equivalent to `Word(")")`).

#### Scenario: Single Unmatched Closing Paren

- **GIVEN** the token array `[ClosingParens]`
- **WHEN** balanceParentheses is applied
- **THEN** the result is `[Phrase(")")]`

#### Scenario: Excess Closing Paren After Balanced Pair

- **GIVEN** the token array `[OpeningParens, Word("some"), ClosingParens, Word("stuff"), ClosingParens]`
- **WHEN** balanceParentheses is applied
- **THEN** the last `ClosingParens` (unmatched) becomes `Phrase(")")`: `[OpeningParens, Word("some"), ClosingParens, Word("stuff"), Phrase(")")]`

### Requirement: Non-Greedy Matching

The system SHALL match parentheses non-greedily, pairing each `OpeningParens` with the nearest subsequent `ClosingParens` at the same nesting level. Adjacent balanced groups `(a)(b)` produce two separate balanced pairs, not one pair spanning the entire sequence.

#### Scenario: Two Adjacent Balanced Groups

- **GIVEN** the token array `[OpeningParens, Word("a"), ClosingParens, OpeningParens, Word("b"), ClosingParens]`
- **WHEN** balanceParentheses is applied
- **THEN** all tokens are returned unchanged as two separate balanced pairs

### Requirement: Nested Balanced Parens

The system SHALL support nested balanced parentheses at arbitrary depth, using O(1) call stack depth regardless of nesting depth.

#### Scenario: Two Levels of Nesting

- **GIVEN** the token array `[OpeningParens, OpeningParens, ClosingParens, ClosingParens]`
- **WHEN** balanceParentheses is applied
- **THEN** all tokens are returned unchanged

#### Scenario: Complex Nested Structure

- **GIVEN** the token array `[OpeningParens, OpeningParens, Word("a"), ClosingParens, Word("b"), OpeningParens, Word("c"), ClosingParens, ClosingParens]`
- **WHEN** balanceParentheses is applied
- **THEN** all tokens are returned unchanged, preserving the outer group containing two inner groups

#### Scenario: Deep nesting does not overflow

- **GIVEN** a token array with 10,000 nested `OpeningParens` followed by a word followed by 10,000 `ClosingParens`
- **WHEN** balanceParentheses is applied
- **THEN** all pairs are preserved without stack overflow
