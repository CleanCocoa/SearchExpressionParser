# Whitespace Handling Specification

> Generated by spec-gen on 2026-04-05
> Source files: `Sources/SearchExpressionParser/Tokenization/Character+isWhitespace.swift`, `Sources/SearchExpressionParser/Tokenization/Tokenizer.swift`, `Sources/SearchExpressionParser/Tokenization/TokenExtractor/WordExtractor.swift`, `Sources/SearchExpressionParser/Tokenization/TokenExtractor/NotExtractor.swift`, `Sources/SearchExpressionParser/Tokenization/TokenExtractor/AndExtractor.swift`, `Sources/SearchExpressionParser/Tokenization/TokenExtractor/OrExtractor.swift`

## Purpose

The `Character.isWhitespace` extension defines the authoritative set of Unicode codepoints treated as whitespace throughout the tokenizer. Whitespace acts as a token separator: it is consumed between tokens, stripped from leading and trailing positions, and collapsed so that consecutive whitespace characters never produce tokens on their own. Whitespace also serves as the required termination condition for keyword operator recognition (AND, OR, NOT).

## Requirements

### Requirement: Whitespace Character Set

The system SHALL recognize exactly the following Unicode codepoints as whitespace, sourced from CoreFoundation `CFUniChar.c`:

| Codepoint | Name |
|-----------|------|
| U+0020 | Space |
| U+0009 | Horizontal Tab |
| U+00A0 | No-Break Space |
| U+1680 | Ogham Space Mark |
| U+2000 | En Quad |
| U+2001 | Em Quad |
| U+2002 | En Space |
| U+2003 | Em Space |
| U+2004 | Three-Per-Em Space |
| U+2005 | Four-Per-Em Space |
| U+2006 | Six-Per-Em Space |
| U+2007 | Figure Space |
| U+2008 | Punctuation Space |
| U+2009 | Thin Space |
| U+200A | Hair Space |
| U+200B | Zero Width Space |
| U+202F | Narrow No-Break Space |
| U+205F | Medium Mathematical Space |
| U+3000 | Ideographic Space |

The `Character.isWhitespace` extension SHALL be the single source of truth for whitespace classification. All other tokenizer components MUST defer to this extension.

#### Scenario: Non-breaking space acts as whitespace

- **GIVEN** an input string containing U+00A0 (No-Break Space) between two words
- **WHEN** the tokenizer produces tokens
- **THEN** U+00A0 SHALL be treated as a token separator, producing two separate `Word` tokens

#### Scenario: Standard space acts as whitespace

- **GIVEN** an input string `"foo bar"` containing U+0020 between words
- **WHEN** the tokenizer produces tokens
- **THEN** the result SHALL be `[Word("foo"), Word("bar")]`

### Requirement: Leading and Trailing Whitespace Stripping

The tokenizer SHALL strip leading and trailing whitespace from the input before producing tokens. Whitespace at the beginning or end of the search string MUST NOT affect the resulting token list.

#### Scenario: Single character with surrounding whitespace

- **GIVEN** an input string `"     x    "`
- **WHEN** the tokenizer produces tokens
- **THEN** the result SHALL be a single `Word("x")` token

#### Scenario: Two words with surrounding whitespace

- **GIVEN** an input string `"   foo   bar   "`
- **WHEN** the tokenizer produces tokens
- **THEN** the result SHALL be `[Word("foo"), Word("bar")]`

### Requirement: Whitespace Collapsing Between Tokens

The tokenizer SHALL treat any sequence of one or more consecutive whitespace characters as a single token boundary. The `skipWhitespace()` method on `TokenCharacterBuffer` consumes all contiguous whitespace characters in a loop before the next token extraction attempt.

#### Scenario: Multiple spaces between words

- **GIVEN** an input string `"   foo   bar   "`
- **WHEN** the tokenizer produces tokens
- **THEN** the result SHALL be exactly two tokens, not empty tokens for intermediate whitespace

### Requirement: Empty and Whitespace-Only Input

The tokenizer SHALL produce zero tokens for empty input and for input consisting entirely of whitespace characters.

#### Scenario: Empty string

- **GIVEN** an empty input string `""`
- **WHEN** the tokenizer produces tokens
- **THEN** the result SHALL be an empty token list

#### Scenario: Single space

- **GIVEN** an input string `" "`
- **WHEN** the tokenizer produces tokens
- **THEN** the result SHALL be an empty token list

#### Scenario: Many spaces

- **GIVEN** an input string `"          "`
- **WHEN** the tokenizer produces tokens
- **THEN** the result SHALL be an empty token list

### Requirement: Operator Keyword Termination by Whitespace

The keyword operators AND, OR, and NOT SHALL only be recognized when followed by a whitespace character. If the character immediately after the keyword letters is not whitespace (or is absent at end-of-input), the keyword MUST be treated as a plain word.

- `NotExtractor` requires `peekNext(3)?.isWhitespace == true` (the character after "NOT")
- `AndExtractor` requires `peekNext(3)?.isWhitespace == true` (the character after "AND")
- `OrExtractor` requires `peekNext(2)?.isWhitespace == true` (the character after "OR")

At end-of-input, `peekNext` returns `nil`, so `nil?.isWhitespace == true` evaluates to `false`, causing the operator to be rejected.

#### Scenario: NOT followed by whitespace

- **GIVEN** an input string `"NOT me"`
- **WHEN** the tokenizer produces tokens
- **THEN** the result SHALL be `[UnaryOperator.not, Word("me")]`

#### Scenario: NOT at end of input (no trailing whitespace)

- **GIVEN** an input string `"NOT"`
- **WHEN** the tokenizer produces tokens
- **THEN** the result SHALL be `[Word("NOT")]`

#### Scenario: NOT with only trailing whitespace

- **GIVEN** an input string `"   \t NOT  "`
- **WHEN** the tokenizer produces tokens
- **THEN** the result SHALL be `[UnaryOperator.not]`

#### Scenario: NOT as prefix of a longer word

- **GIVEN** an input string `"NOTest"`
- **WHEN** the tokenizer produces tokens
- **THEN** the result SHALL be `[Word("NOTest")]`

#### Scenario: AND at end of input

- **GIVEN** an input string `"AND"`
- **WHEN** the tokenizer produces tokens
- **THEN** the result SHALL be `[Word("AND")]`

#### Scenario: AND as prefix of a longer word

- **GIVEN** an input string `"ANDromeda"`
- **WHEN** the tokenizer produces tokens
- **THEN** the result SHALL be `[Word("ANDromeda")]`

#### Scenario: OR at end of input

- **GIVEN** an input string `"OR"`
- **WHEN** the tokenizer produces tokens
- **THEN** the result SHALL be `[Word("OR")]`

#### Scenario: OR as prefix of a longer word

- **GIVEN** an input string `"ORwell"`
- **WHEN** the tokenizer produces tokens
- **THEN** the result SHALL be `[Word("ORwell")]`

### Requirement: Word Termination by Whitespace

The `WordExtractor` SHALL consume characters until it encounters a whitespace character (as defined by `Character.isWhitespace`), a parenthesis, a quotation mark, or end-of-input. Whitespace thus serves as the primary word boundary delimiter.

#### Scenario: Sentence tokenization

- **GIVEN** an input string `"Lorem ipsum dolor sit amet!"`
- **WHEN** the tokenizer produces tokens
- **THEN** each whitespace-separated run of non-whitespace characters SHALL become a separate `Word` token

### Requirement: Whitespace Inside Quoted Phrases Is Preserved

Whitespace within quotation marks SHALL NOT be stripped or collapsed. The `PhraseExtractor` preserves all characters (including whitespace) between opening and closing quotation marks verbatim.

#### Scenario: Quoted phrase with internal whitespace

- **GIVEN** an input string `"\"  fair   play \""`
- **WHEN** the tokenizer produces tokens
- **THEN** the result SHALL be `[Phrase("  fair   play ")]` with internal whitespace preserved
