# Expression Evaluation Specification

> Generated by spec-gen on 2026-04-05
> Source files: Sources/SearchExpressionParser/Parsing/Expressions.swift, Tests/SearchExpressionParserTests/ExpressionTests.swift

## Purpose

The Expression evaluation system defines a tree of nodes that can test whether a string satisfies a search expression. Each node implements two evaluation paths: `StringExpressionSatisfiable` for Swift String matching and `CStringExpressionSatisfiable` for high-performance C-string matching. Composite nodes (And, Or, Not) combine leaf nodes (Anything, Contains) using boolean logic with short-circuit evaluation.

## Requirements

### Requirement: Dual Evaluation Paths

The system SHALL provide two protocols for expression evaluation. `StringExpressionSatisfiable` requires a `contains(phrase:)` method accepting a Swift `String`. `CStringExpressionSatisfiable` requires a `matches(needle:)` method accepting a `ContainsNode.CString` (which is `[CChar]`). Every `Expression` node MUST implement `isSatisfied(by:)` for both protocols.

#### Scenario: String conformance on Swift String

- **GIVEN** a Swift `String` value
- **WHEN** evaluated via the `StringExpressionSatisfiable` path
- **THEN** `contains(phrase:)` SHALL delegate to Swift's standard `String.contains(_:)`, which performs case-sensitive substring matching

### Requirement: AnythingNode Always Matches

`AnythingNode` SHALL return `true` for any input, regardless of evaluation path. It acts as a wildcard expression.

#### Scenario: AnythingNode with empty string

- **GIVEN** an `AnythingNode`
- **WHEN** evaluated against an empty string `""`
- **THEN** the result SHALL be `true`

#### Scenario: AnythingNode with arbitrary content

- **GIVEN** an `AnythingNode`
- **WHEN** evaluated against any non-empty string (including whitespace-only and newline-only strings)
- **THEN** the result SHALL be `true`

### Requirement: ContainsNode Substring Match

`ContainsNode` SHALL perform substring matching against the input. Via the `StringExpressionSatisfiable` path, it delegates to the satisfiable's `contains(phrase:)` method, passing its stored `string` property. Via the `CStringExpressionSatisfiable` path, it delegates to the satisfiable's `matches(needle:)` method, passing its precomputed `cString` property.

#### Scenario: Substring found

- **GIVEN** a `ContainsNode("e")`
- **WHEN** evaluated against `"something"`
- **THEN** the result SHALL be `true`

#### Scenario: Multi-character substring found

- **GIVEN** a `ContainsNode("et")`
- **WHEN** evaluated against `"something"`
- **THEN** the result SHALL be `true`

#### Scenario: Substring not found

- **GIVEN** a `ContainsNode("a")`
- **WHEN** evaluated against `"something"`
- **THEN** the result SHALL be `false`

#### Scenario: Substring with space not found

- **GIVEN** a `ContainsNode("m t")`
- **WHEN** evaluated against `"something"`
- **THEN** the result SHALL be `false`

### Requirement: ContainsNode Empty String Behavior

When a `ContainsNode` is initialized with an empty string `""`, it SHALL return `false` for all inputs via the `StringExpressionSatisfiable` path (using the default `String` conformance). This includes empty strings, whitespace, newlines, and arbitrary content.

#### Scenario: Empty search term against empty input

- **GIVEN** a `ContainsNode("")`
- **WHEN** evaluated against `""`
- **THEN** the result SHALL be `false`

#### Scenario: Empty search term against non-empty input

- **GIVEN** a `ContainsNode("")`
- **WHEN** evaluated against `"something"`
- **THEN** the result SHALL be `false`

### Requirement: ContainsNode CString Factory

`ContainsNode` SHALL expose a `static var cStringFactory` property of type `(String) -> CString`. The default factory applies `precomposedStringWithCanonicalMapping`, then `lowercased()`, then encodes as UTF-8 via `cString(using: .utf8)`, falling back to an empty array on encoding failure. Consumers MAY replace `cStringFactory` to customize CString creation for all subsequent `ContainsNode` initializations.

#### Scenario: Default CString creation

- **GIVEN** the default `cStringFactory`
- **WHEN** a `ContainsNode` is initialized with a string
- **THEN** the `cString` property SHALL contain the lowercased, precomposed canonical mapping of that string encoded as UTF-8

### Requirement: AndNode Short-Circuit Evaluation

`AndNode` SHALL evaluate its `lhs` expression first. If `lhs` returns `false`, the result SHALL be `false` without evaluating `rhs`. If `lhs` returns `true`, it SHALL evaluate `rhs` and return that result. This applies to both evaluation paths. The implementation SHALL use O(1) call stack depth via iterative evaluation.

#### Scenario: Both true

- **GIVEN** an `AndNode` with truthy lhs and truthy rhs
- **WHEN** evaluated
- **THEN** the result SHALL be `true`

#### Scenario: Left false, right true

- **GIVEN** an `AndNode` with falsy lhs and truthy rhs
- **WHEN** evaluated
- **THEN** the result SHALL be `false`

#### Scenario: Left true, right false

- **GIVEN** an `AndNode` with truthy lhs and falsy rhs
- **WHEN** evaluated
- **THEN** the result SHALL be `false`

#### Scenario: Both false

- **GIVEN** an `AndNode` with falsy lhs and falsy rhs
- **WHEN** evaluated
- **THEN** the result SHALL be `false`

### Requirement: OrNode Short-Circuit Evaluation

`OrNode` SHALL evaluate its `lhs` expression first. If `lhs` returns `true`, the result SHALL be `true` without evaluating `rhs`. If `lhs` returns `false`, it SHALL evaluate `rhs` and return that result. This applies to both evaluation paths. The implementation SHALL use O(1) call stack depth via iterative evaluation.

#### Scenario: Both true

- **GIVEN** an `OrNode` with truthy lhs and truthy rhs
- **WHEN** evaluated
- **THEN** the result SHALL be `true`

#### Scenario: Left false, right true

- **GIVEN** an `OrNode` with falsy lhs and truthy rhs
- **WHEN** evaluated
- **THEN** the result SHALL be `true`

#### Scenario: Left true, right false

- **GIVEN** an `OrNode` with truthy lhs and falsy rhs
- **WHEN** evaluated
- **THEN** the result SHALL be `true`

#### Scenario: Both false

- **GIVEN** an `OrNode` with falsy lhs and falsy rhs
- **WHEN** evaluated
- **THEN** the result SHALL be `false`

### Requirement: NotNode Boolean Negation

`NotNode` SHALL return the boolean negation of its inner expression's evaluation result. This applies to both evaluation paths. The implementation SHALL use O(1) call stack depth via iterative evaluation.

#### Scenario: Negate true

- **GIVEN** a `NotNode` wrapping a truthy expression
- **WHEN** evaluated
- **THEN** the result SHALL be `false`

#### Scenario: Negate false

- **GIVEN** a `NotNode` wrapping a falsy expression
- **WHEN** evaluated
- **THEN** the result SHALL be `true`
